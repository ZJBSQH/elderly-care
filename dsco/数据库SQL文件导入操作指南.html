数据库SQL文件导入操作指南

项目信息：
项目名称：elderly_care_cloud（智慧养老云平台）
数据库类型：MySQL 8.0
数据库端口：13306（宿主机映射端口）
SQL文件位置：init-sql 目录

一、环境准备

1.1 启动Docker容器

在导入SQL文件之前，需要确保MySQL容器已经启动。在项目根目录下执行以下命令：

docker-compose up -d

查看容器运行状态：

docker ps
预期结果：应该看到名为 elderly-mysql 的容器处于运行状态，端口映射为 0.0.0.0:13306->3306/tcp

1.2 SQL文件清单

项目的 init-sql 目录包含以下SQL文件：

序号 文件名 说明
1 db_elderly_auth.sql 认证授权模块数据库
2 db_elderly_admin.sql 后台管理模块数据库
3 db_elderly_health.sql 健康管理模块数据库
4 db_elderly_medicine.sql 用药管理模块数据库
5 db_elderly_news.sql 新闻资讯模块数据库
6 db_elderly_remind.sql 提醒服务模块数据库
7 db_elderly_user.sql 用户管理模块数据库

二、IDEA Database面板配置

2.1 打开Database面板

1在IDEA右侧边栏找到 "Database" 标签并点击

2如果没有看到，点击菜单栏 View → Tool Windows → Database

2.2 创建MySQL连接

1点击Database面板左上角的 "+" 号

2选择 "Data Source" → "MySQL"

3在弹出的配置窗口中填写以下信息：

配置项 配置值 说明
Name elderly_mysql 连接名称（可自定义）
Host localhost 数据库主机地址
Port 13306 端口号（注意：docker映射端口）
User root 用户名
Password Zheng666 密码
Database （留空） 数据库名称（导入时留空）
重要提示：端口号必须填写 13306，而不是默认的 3306。因为docker-compose.yml中将容器内部的3306端口映射到了宿主机的13306端口。

2.3 测试连接

1点击 "Test Connection" 按钮

2如果提示缺少驱动，点击 "Download missing driver files" 自动下载

3测试成功后会显示绿色的对勾图标

4点击 "OK" 保存连接配置

成功标志：Database面板中出现 elderly_mysql 连接，左侧有绿色圆点表示连接正常。

三、导入SQL文件

3.1 方法一：直接运行SQL脚本（推荐）

1在Database面板中，右键点击 elderly_mysql 连接

2选择 "Run SQL Script..."

3在文件选择对话框中，导航到项目的 init-sql 目录

4选择要导入的SQL文件（例如：db_elderly_auth.sql）

5点击 "Run" 按钮执行

6在底部 "Messages" 面板查看执行结果

3.2 方法二：在SQL编辑器中执行

1在IDEA项目中，展开 init-sql 目录

2双击打开某个SQL文件（例如：db_elderly_auth.sql）

3在SQL编辑器上方，找到绿色三角形运行按钮

4点击运行按钮旁边的下拉菜单，选择 "Run on 'elderly_mysql'"

5点击绿色运行按钮执行整个脚本

3.3 方法三：使用Query Console批量导入

1在Database面板中，右键点击 elderly_mysql 连接

2选择 "New" → "Query Console"

3在打开的控制台中输入以下命令：

source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_auth.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_admin.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_health.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_medicine.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_news.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_remind.sql;
source F:/Workspace/JAVA_Workspace/elderly_care_cloud/init-sql/db_elderly_user.sql;

4点击运行按钮执行所有脚本

建议导入顺序：
1. db_elderly_auth.sql（认证授权，基础模块）
2. db_elderly_user.sql（用户管理，依赖认证）
3. 其他模块SQL文件（无特定顺序要求）

四、验证导入结果

4.1 刷新数据库连接

1在Database面板中，右键点击 elderly_mysql 连接

2选择 "Refresh" 或按快捷键 Ctrl+F5

4.2 查看数据库和表结构

1展开 elderly_mysql 连接

2展开 schemas

3应该看到新创建的数据库（如：elderly_auth、elderly_user 等）

4继续展开数据库，查看 tables 下的表结构

4.3 查看表数据

1右键点击某个表

2选择 "View Data" → "View Table"

3在右侧面板中查看表数据

导入成功的标志：
- 所有数据库都已创建
- 每个数据库下都有对应的表
- 表中有初始数据（如果有）

五、常见问题及解决方案

5.1 连接失败

问题：Test Connection 失败，提示连接超时或拒绝连接

解决方案：
1. 检查Docker容器是否运行：docker ps
2. 确认端口配置正确：必须是 13306
3. 检查防火墙设置，确保13306端口未被阻止
4. 重启MySQL容器：docker-compose restart mysql

5.2 执行SQL报错

问题：执行SQL文件时提示语法错误或数据库不存在

解决方案：
1. 检查SQL文件编码是否为 UTF-8
2. 手动创建数据库：
CREATE DATABASE IF NOT EXISTS elderly_auth DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
3. 在SQL文件开头添加：USE elderly_auth;

5.3 找不到Database面板

问题：IDEA中没有Database面板

解决方案：
1. 确认IDEA版本：社区版（Community）可能没有此功能
2. 安装插件：Settings → Plugins → 搜索 "Database Navigator"
3. 使用命令行导入：docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_auth.sql

5.4 驱动下载失败

问题：下载MySQL驱动失败

解决方案：
1. 检查网络连接
2. 配置IDEA代理：Settings → Appearance & Behavior → System Settings → HTTP Proxy
3. 手动下载驱动并添加：Settings → Database → Drivers → MySQL

六、备选方案：命令行导入

如果IDEA Database面板无法使用，可以通过命令行导入：

docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_admin.sql
docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_health.sql
docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_medicine.sql
docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_news.sql
docker exec -i elderly-mysql mysql -uroot -pZheng666 < db_elderly_remind.sql

6.2 使用mysql客户端

mysql -h 127.0.0.1 -P 13306 -u root -pZheng666 < db_elderly_auth.sql

七、总结

导入流程总结：
1. 启动Docker容器（docker-compose up -d）
2. 打开IDEA Database面板
3. 创建MySQL连接（端口13306）
4. 测试连接并保存
