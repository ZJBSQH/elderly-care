三、导入数据库

重要：数据库会在 Docker 容器首次启动时自动导入 init-sql/ 目录下的 SQL 文件。
如果自动导入失败，请手动导入。

3.1 自动导入（推荐）

docker-compose.yml 已配置自动挂载 init-sql 目录，首次启动时会自动执行：

# 查看数据库是否已创建
docker exec -it elderly-mysql mysql -uroot -pZheng666 -e "SHOW DATABASES;"

应该看到以下数据库：

  • db_elderly_admin
  • db_elderly_auth
  • db_elderly_health
  • db_elderly_medicine
  • db_elderly_news
  • db_elderly_remind
  • db_elderly_user
  • nacos_config（Nacos 配置中心数据库）

3.2 手动导入（备选）

如果自动导入失败，使用 IDEA Database 面板手动导入：

  1. 打开 IDEA 右侧 "Database" 面板
  2. 创建 MySQL 连接（Host: localhost, Port: 13306, User: root, Password: Zheng666）
  3. 右键连接 → "Run SQL Script"
  4. 依次选择 init-sql/ 目录下的 SQL 文件执行

四、配置 Nacos

4.1 访问 Nacos 控制台

# 浏览器访问
http://localhost:8848/nacos

# 登录
用户名：nacos

4.2 创建命名空间（可选）

  1. 进入 "命名空间" 菜单
  2. 点击 "新建命名空间"
  3. 填写：命名空间ID: dev，命名空间名: 开发环境

4.3 创建配置文件

每个微服务都需要在 Nacos 中创建配置文件，格式为 {spring.application.name}.yaml

配置示例（elderly-auth.yaml）：
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://elderly-mysql:3306/db_elderly_auth?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: Zheng666
  
  redis:
    host: elderly-redis
    port: 6379
    database: 0

server:
  port: 8081

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
注意：如果使用本地开发环境，需要将配置中的 elderly-mysql 改为 localhost，端口改为 13306；elderly-redis 改为 localhost

八、常见问题

8.1 服务启动失败

问题：服务启动时报错 "Unable to connect to Nacos Server"
解决方案：
1. 检查 Nacos 是否启动：docker ps | grep nacos
2. 检查端口是否开放：curl http://localhost:8848/nacos
3. 检查 bootstrap.yml 中的 Nacos 地址配置
问题：服务启动时报错 "Communications link failure"（MySQL 连接失败）
解决方案：
1. 检查 MySQL 是否启动：docker ps | grep mysql
2. 检查数据库是否创建：docker exec -it elderly-mysql mysql -uroot -pZheng666 -e "SHOW DATABASES;"
3. 检查 Nacos 配置中的数据库连接信息
问题：服务启动时报错 "Unable to connect to Redis"
解决方案：
1. 检查 Redis 是否启动：docker ps | grep redis
2. 测试 Redis 连接：docker exec -it elderly-redis redis-cli ping
3. 检查 Nacos 配置中的 Redis 连接信息

8.2 Gateway 路由失败

问题：访问 Gateway 报错 "503 Service Unavailable"
解决方案：
1. 检查目标服务是否启动并注册到 Nacos
2. 检查 Nacos 控制台的服务列表
3. 检查 Gateway 的路由配置
4. 查看 Gateway 日志，确认路由规则是否正确

8.3 JWT 认证失败

问题：接口返回 401 Unauthorized
解决方案：
1. 确认请求 Header 中携带了 Authorization: Bearer {token}
2. 检查 token 是否过期
3. 检查 JWT 密钥配置是否一致（各服务的 jwt.secret）
4. 查看服务日志，确认 JWT 解析是否成功

8.4 跨域问题

问题：前端调用接口报错 "CORS policy"
解决方案：
Gateway 已配置全局 CORS，允许所有来源。如果仍有问题：
1. 检查 Gateway 的 CORS 配置
2. 确认请求是通过 Gateway 发送，而不是直接访问服务
3. 检查浏览器控制台的详细错误信息

8.5 数据库连接池耗尽

问题：服务运行一段时间后报错 "Connection pool exhausted"
解决方案：
1. 在 Nacos 配置中增加连接池大小：
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
2. 检查是否有未关闭的数据库连接
3. 重启服务释放连接

8.6 Docker 容器启动失败

问题：docker-compose up 报错 "port is already allocated"
解决方案：
1. 检查端口占用：netstat -ano | findstr "13306"
2. 停止占用端口的进程或修改 docker-compose.yml 中的端口映射
3. 停止之前的容器：docker-compose down
