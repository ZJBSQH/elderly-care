# AI用药助手功能说明

## 📋 功能概述

AI用药助手是基于Spring AI实现的智能用药辅助功能，提供用药咨询、智能提醒、漏服干预等服务。

---

## ✨ 核心功能

### 1️⃣ 智能问答（💬）
- **功能描述**：老人或家属可以输入用药相关问题，AI基于老人的用药计划、历史记录等信息生成个性化回答
- **使用场景**：
  - "阿司匹林什么时候吃？"
  - "这个药有什么副作用？"
  - "忘记吃药怎么办？"
  - "需要忌口吗？"
- **技术实现**：
  - 后端调用 `MedicineContextBuilder` 构建上下文
  - 通过 `MedicineAIProvider` 调用AI模型
  - 支持3次重试机制

### 2️⃣ 今日建议（📋）
- **功能描述**：AI分析老人今日用药计划，生成综合建议和注意事项
- **使用场景**：
  - 查看今日用药整体安排
  - 了解特殊注意事项
  - 获取饮食建议
- **特点**：一键刷新，实时生成

### 3️⃣ 用药提醒（🔔）
- **功能描述**：手动触发AI生成个性化用药提醒，通过WebSocket推送给老人
- **使用场景**：
  - 家属远程发起提醒
  - 定时提醒补充
- **特点**：语气温和，适合老年人

### 4️⃣ 漏服干预（⚠️）
- **功能描述**：当检测到老人漏服药物时，AI生成干预消息并提供补服建议
- **使用场景**：
  - 标记任务为漏服
  - AI生成干预建议
  - 提醒联系医生或家属

## 🔧 技术架构

### 后端结构
```
com.elderlycare.ai/
├── context/
│   └── MedicineContextBuilder.java    # 上下文构建器
├── provider/
│   └── MedicineAIProvider.java        # AI服务提供者
├── notification/
│   └── MedicineNotifier.java          # 消息通知器
└── orchestration/
    └── AIAssistantServiceImpl.java    # 服务编排层
```

### 前端结构
```
elder_care/
├── api/
│   └── ai-assistant.js                # API封装
└── pages/medicine/
    └── ai-assistant.vue               # AI助手页面
```

### API接口
| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用药问答 | POST | `/ai-assistant/medicine/question` | elderId + question |
| 发起提醒 | POST | `/ai-assistant/medicine/remind/{elderId}` | 路径参数elderId |
| 今日建议 | GET | `/ai-assistant/medicine/today-advice/{elderId}` | 路径参数elderId |
| 漏服干预 | POST | `/ai-assistant/medicine/missed` | elderId + taskId |

## 🚀 使用流程

### 第一步：进入AI助手
1. 打开"今日用药"页面
2. 点击右上角 🤖 AI助手按钮
3. 或在页面列表中找到"AI用药助手"

### 第二步：选择老人
- 在顶部下拉框选择要咨询的老人
- 系统会自动加载该老人的用药计划

### 第三步：使用功能

#### 💬 智能问答
1. 切换到"智能问答"标签
2. 在输入框输入问题，或点击快捷问题
3. 等待AI回复（通常5-15秒）
4. 查看对话历史

#### 📋 今日建议
1. 切换到"今日建议"标签
2. 点击"获取建议"按钮
3. 查看AI生成的今日用药建议
4. 可随时点击"刷新"更新建议

#### 🔔 用药提醒
1. 切换到"用药提醒"标签
2. 点击"发起用药提醒"按钮
3. 系统通过WebSocket推送提醒消息
4. 可在"漏服干预"区域标记漏服任务

## ⚙️ 配置说明

### 后端配置（application.yml）
```yaml
# AI超时时间（秒）
ai:
  timeout: 30
  max-retries: 3

# Spring AI配置
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-plus
```

### 前端配置
- 无需额外配置
- 确保已登录并获取Token
- WebSocket连接地址在 `NotifyWebSocket` 中配置

## 🔍 常见问题

### Q1: AI响应很慢怎么办？
A: 
- 正常响应时间为5-15秒
- 检查网络连接
- 后端有3次重试机制
- 可考虑添加响应缓存

### Q2: 如何选择老人？
A:
- 从登录用户的 `userInfo.elders` 中获取
- 目前使用模拟数据，需对接真实接口

### Q3: WebSocket收不到消息？
A:
- 检查WebSocket连接状态
- 确认elderId正确
- 查看浏览器控制台错误信息
- 降级方案：使用轮询

### Q4: AI回答不准确？
A:
- 检查用药计划是否完整
- 优化Prompt提示词
- 调整AI模型参数
- 考虑更换更强的模型
