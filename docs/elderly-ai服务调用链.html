链路一：普通 AI 问答（旧链路）

触发入口：POST /ai/medicine/question 或 GET /ai/medicine/today-advice

调用链

AssistantController │ ├─ MedicalContextBuilder.buildAndFormat(elderId) │ │ │ └─ MedicalContextBuilderImpl.buildAndFormat(elderId) │ │ │ ├─ MedicineFeignClient.getMedicinesByElderId(elderId) │ │ └─ HTTP GET /medicine/look/{elderId} │ │ └─ elderly-medicine :8084 → 药品清单 │ │ │ ├─ MedicineFeignClient.getTodayRecords(elderId) │ │ └─ HTTP GET /record/today?elderId=x │ │ └─ elderly-medicine :8084 → 今日用药记录 │ │ │ ├─ RemindFeignClient.getTodayTasks() │ │ └─ HTTP GET /remind/task/today │ │ └─ elderly-remind :8085 → 今日提醒任务 │ │ │ └─ formatToNaturalLanguage() → 转自然语言文本 │ └─ MedicalAssistant.chat(fullMessage) └─ DashScope ChatModel (qwen3.6-flash) └─ 阿里云 DashScope API → Flux<String> SSE 流

链路二：RAG 增强问答（新链路）

触发入口：POST /ai/rag/ask 或 POST /ai/rag/health-knowledge

2.1 完整调用链

RagAssistantController.askWithRag(dto) │ └─ RagService.askWithRag(elderId, question) │ ├─❶ RagService.retrieveRelevant(question) [private] │ │ │ ├─ EmbeddingModel.embed(question) │ │ └─ 千问 DashScope text-embedding-v3 → 1536维向量 │ │ │ ├─ new EmbeddingSearchRequest(queryEmbedding, maxResults, minScore) │ │ │ └─ RedisEmbeddingStore.search(request) │ │ │ ├─ embeddingToBytes() → FLOAT32 小端 6144字节 │ │ │ ├─ Jedis.ftSearch(indexName, query, params) │ │ └─ Redis Stack (RediSearch) → HNSW KNN │ │ │ └─ 遍历结果: │ ├─ getProperty(doc, "__score") → 距离转相似度 │ ├─ blobToEmbedding() → 反序列化向量 │ └─ buildMetadata() → 组装 LangChain4j Metadata │ ├─❷ MedicalContextBuilder.buildAndFormat(elderId) │ └─ (复用链路一，拉取个人用药数据) │ └─ elderId==null 时跳过，返回 "(无个人数据)" │ ├─❸ RagService.buildAugmentedPrompt(docs, ctx, question) [private] │ └─ 组装: System提示 + 参考知识 + 个人数据 + 用户问题 │ └─❹ MedicalAssistant.chat(augmentedPrompt) └─ DashScope ChatModel └─ 阿里云 DashScope API → Flux<String> SSE 流

2.2 降级分支（检索失败时）

RagService.askWithRag() → catch 异常 │ ├─ elderId != null → MedicalContextBuilder.buildAndFormat(elderId) ├─ elderId == null → "(无个人数据)" │ └─ MedicalAssistant.chat(fallbackPrompt) → 纯大模型回答，无知识库

2.3 依赖关系矩阵

RagService ├── EmbeddingModel (千问 DashScope, LangChain4j 自动配置) ├── RedisEmbeddingStore (自定义, 在 RagConfig 中装配) ├── MedicalAssistant (@AiService, 复用旧链路) └── MedicalContextBuilder (复用旧链路)

链路三：知识库管理

3.1 上传文档

触发入口：POST /ai/kb/document

KnowledgeBaseController.uploadDocument(dto) │ └─ KnowledgeService.uploadDocument(dto) │ ├─ 校验分类: ALLOWED_CATEGORIES.contains(category) │ └─ IngestionPipeline.ingest(text, metadata) │ ├─ 阶段1: split(text, metadata) │ ├─ DocumentSplitters.recursive(500, 50) │ ├─ splitter.split(Document.from(text)) │ └─ 为每段注入 title/category/doc_id/chunk_index 元数据 │ ├─ 阶段2: embed(segments) │ └─ EmbeddingModel.embedAll(segments) │ └─ 千问 DashScope text-embedding-v3 → 1536维向量 × N │ └─ 阶段3: store └─ RedisEmbeddingStore.addAll(embeddings, segments) └─ 逐个 addInternal(id, embedding, segment) ├─ embeddingToBytes() → FLOAT32 小端字节 ├─ bytesToBlob() → ISO-8859-1 字符串 └─ Jedis.hset(key, fields) └─ Redis Stack → RediSearch 自动索引

3.2 删除文档

触发入口：DELETE /ai/kb/document/{id}

KnowledgeBaseController.deleteDocument(id) │ └─ KnowledgeService.deleteDocument(docId) │ └─ RedisEmbeddingStore.removeByDocId(docId) │ ├─ Jedis.ftSearch(indexName, "@doc_id:{docId}") │ └─ Redis Stack RediSearch → 找到所有匹配 Key │ └─ Jedis.del(keys) └─ Redis Stack → 批量删除 Hash + 自动移除索引

3.3 更新文档

触发入口：PUT /ai/kb/document/{id}

KnowledgeBaseController.updateDocument(id, dto) │ └─ KnowledgeService.updateDocument(docId, dto) │ ├─ RedisEmbeddingStore.removeByDocId(docId) │ └─ (同 3.2 删除链路) │ └─ IngestionPipeline.ingest(text, metadata) └─ (同 3.1 上传链路，复用 docId)

3.4 批量上传

KnowledgeBaseController.uploadDocuments(dtos) │ └─ KnowledgeService.uploadDocuments(dtos) │ └─ for each dto: └─ KnowledgeService.uploadDocument(dto) → 同 3.1

3.5 依赖关系矩阵

KnowledgeService ├── IngestionPipeline (文档摄取) └── RedisEmbeddingStore (向量存储, 用于删除) IngestionPipeline ├── EmbeddingModel (千问 DashScope) └── RedisEmbeddingStore (向量存储, 用于写入) RedisEmbeddingStore └── JedisPooled (Jedis 6.x → Redis Stack RediSearch)

初始化链：容器启动时

Spring Boot 启动 │ ├─ RagConfig → 创建 JedisPooled Bean │ └─ 连接 Redis Stack (host:port, 读取 spring.data.redis.*) │ └─ RagConfig → 创建 RedisEmbeddingStore Bean └─ new RedisEmbeddingStore(jedis, indexName, prefix, dimension) └─ ensureIndexExists() ├─ Jedis.ftInfo(indexName) → 成功 → "索引已存在" └─ Jedis.ftInfo(indexName) → 异常 └─ createIndex() └─ Jedis.ftCreate(indexName, options, schema) └─ Redis Stack → HNSW 向量索引就绪
